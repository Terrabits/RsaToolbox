Detailed Description

Connects to an intrument via Tcp socket over Ethernet or WiFi.

Warning: TcpBus relies on QTcpSocket, which has well-documented, long-standing reliability issues on Windows platforms. If you are deploying to Windows, consider using VisaBus instead. Mac OS X, Linux, iOS and Android should work well.

Note
TcpBus only supports TCPIP_CONNECTION methods

bool local ( )
slot

Tcp does not have a remote/local mechanism.

This method does nothing and returns false.

You may consider using the SCPI commands "@LOC" and "@REM", so long as they are supported by the instrument. These commands are supported, for example, by Rohde & Schwarz VNAs.

You may also consider using a bus that supports this feature, such as VisaBus.

Returns
false
