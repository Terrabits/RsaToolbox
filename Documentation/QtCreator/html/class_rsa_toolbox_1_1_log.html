Detailed Description

Log creates and manages a log file containing a list of SCPI commands

The Log class can be attached to various GenericBus and GenericInstrument subclasses for automatic documentation of SCPI commands. Log can also be written to manually.

Here is an example of a Log being used in conjunction with a GenericInstrument:

// Log file: "C:\Test Log.txt"
// App name: "My App"
// Version: 1.0.0
Log log("C:\", "Test Log.txt", "My App", "1.0.0");
// Instrument connection
// Method: GPIB
// Address: 17
// Timeout: 1000 ms (1 second)
GenericInstrument fsw(GPIB_CONNECTION, "17", 1000);
See also
GenericInstrument::useLog(), GenericBus::print()

Constructor & Destructor Documentation

Log ( QObject *  parent = 0)
explicit

The default constructor.

This constructor is provided for compatibility in instances where a default constructor is important, for example when used as a property in a class.

Instances created with this constructor are not meant to be used. Such instances will return isOpen() == false or isClosed() == true. Any other method calls (such as print) may cause the application to crash.

Constructor for creating a fully-functional Log object.

This constructor will create a log file in location directory with name filename. For the purposes of writing a header to the log, applicationName and applicationVersion will be used.

A call to isOpen() or isClosed() should be made to confirm that the log file was successfully created.
