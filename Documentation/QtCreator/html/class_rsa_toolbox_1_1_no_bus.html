bool read ( char *  buffer,

virtual

Reads from instrument using c-style character buffer.

This method is pure virtual and must be overloaded in base class implementations. This function should call printRead internally.

This function assumes that the read value is a c-style string that is null terminated. As such, it is assumed that the data to be read does not contain invalid or reserved values (such as the null character). binaryRead does not make this assumption.

A failed read should return false and return an empty c-string:

buffer[0] = '\0';
Note
GenericBus uses this function, along with an internal buffer, to implement other higher-level read functions.
Parameters
bufferC-style character buffer
bufferSize_BSize of buffer, in bytes
Returns
true if read is successful

bool write ( QString  scpi)
virtual

Writes scpi to the connected instrument.

This method is pure virtual and must be overloaded in base class implementations.

If in implementing this function a C-style const char* representation of scpi is necessary (in a US-ASCII/Latin-1 character set) it can be retrieved by calling:

scpi.toLatin1().constData()

It is important to recognize that the above pointer only exists ephemerally, during the execution of this line of code. This is because const char* is pointing to an internal buffer in QByteArray, which disappears after this line is executed.

If you need a persistent pointer, use the verbose snippet below:

QByteArray byteArray = scpi.toLatin1();
const char* cstring = byteArray.constData();
Note
bufferSize_B does not limit write comamnds, as the internal buffer is not used.

true if successul
